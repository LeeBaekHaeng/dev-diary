# 4. 전체 프로젝트 구조

```text
src
 ├─ main
 │   ├─ java
 │   ├─ resources
 │   └─ webapp
 │
 └─ test
     └─ java
     └─ resources
```

### 핵심 관점

* java → 비즈니스 로직
* resources → 설정 및 SQL
* webapp → 화면(View)
* test → 단위 테스트

---

# 5. Java 패키지 구조

```text
egovframework
 └─ let
      ├─ cop
      ├─ sym
      ├─ sec
      ├─ sts
      └─ ...
```

## 패키지 의미

| 패키지 | 설명     |
| --- | ------ |
| cop | 공통업무   |
| sym | 시스템관리  |
| sec | 보안     |
| sts | 통계     |

---

# 6. 공통 컴포넌트 분류

## COP (Common Operation)

대표 기능

* 게시판
* 댓글
* 커뮤니티
* 동호회

예시

```text
cop
 └─ bbs
```

---

# 7. 게시판(BBS) 구조 분석

```text
bbs
 ├─ web
 ├─ service
 └─ service.impl
```

### 가장 많이 사용되는 예제

게시판은 전형적인 전자정부 표준프레임워크 구조를 모두 포함

# 8. 계층형 아키텍처

```text
Controller
    ↓
Service
    ↓
DAO
    ↓
Database
```

### 특징

* 역할 분리
* 테스트 용이
* 유지보수 편리

---

# 9. Controller 계층

예시

```java
@Controller
public class BBSManageController {
}
```

### 역할

* 요청 수신
* 파라미터 처리
* Service 호출
* View 반환

---

# 10. Service 계층

예시

```java
public interface BBSManageService {
}
```

```java
@Service
public class BBSManageServiceImpl {
}
```

### 역할

* 비즈니스 로직 처리
* 트랜잭션 관리
* DAO 호출

---

# 11. DAO 계층

예시

```java
@Repository
public class BBSManageDAO {
}
```

### 역할

* SQL 실행
* 데이터 조회
* 데이터 저장

---

# 12. Mapper(XML)

```text
resources
 └─ egovframework
      └─ mapper
```

예시

```xml
<select id="selectBoardList">
```

### 역할

* SQL 관리
* DB 독립성 확보

---

# 13. VO(Value Object)

예시

```java
BoardVO
BoardMasterVO
ArticleVO

extends ComDefaultVO
```

### 역할

* 데이터 전달 객체
* 검색 조건 저장
* 화면 데이터 전달

---

# 14. Domain 객체

Board
BoardMaster

### 특징

# 22. 실무 분석 순서

신규 프로젝트 분석 시

```text
1. Controller 확인
2. Service 확인
3. DAO 확인
4. Mapper 확인
5. VO 확인
6. 화면 확인
```

가장 빠르게 구조를 파악할 수 있는 방법

# 23. 요약

## 기억해야 할 5가지

1. Spring 기반 계층형 구조
2. Controller → Service → DAO
3. SQL은 Mapper XML 관리
4. VO 중심 데이터 전달
5. 공통컴포넌트는 실무 참고서 역할
